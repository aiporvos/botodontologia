# Dental React UI

Usa esta skill para generar pantallas, dashboards, layouts, sidebars, tablas, cards, métricas y experiencias visuales premium para un sistema odontológico.

## Objetivo visual

La interfaz debe verse:

- premium
- limpia
- moderna
- médica pero no fría
- corporativa
- clara para uso administrativo intensivo

## Lineamientos visuales obligatorios

- React + TypeScript
- Diseño modular por componentes
- Layout de producto SaaS profesional
- Sidebar elegante
- Header liviano con acciones rápidas
- Cards con jerarquía visual fuerte
- Tablas limpias, legibles y con filtros
- Formularios claros y espaciados
- Estados vacíos bien diseñados
- Skeleton loaders
- Badges para estados
- Gráficos simples y ejecutivos
- Buen uso de whitespace

## Estilo visual deseado

- Bordes suaves
- Sombras sutiles
- Alto contraste útil, sin verse agresivo
- Colores clínicos modernos
- Excelente tipografía y escala
- Prioridad de lectura inmediata

## Diseño de dashboard

Siempre que diseñes un dashboard incluir:

- KPIs principales
- agenda del día
- pacientes recientes
- próximos turnos
- métricas de confirmación/cancelación
- ingresos o cobranzas
- alertas operativas
- actividad reciente
- accesos rápidos a acciones comunes

## UX obligatoria

- Cada pantalla debe tener un objetivo claro
- Minimizar fricción administrativa
- Evitar saturar con información
- Las acciones primarias deben destacar
- Las acciones destructivas deben estar protegidas
- Preparar diseño responsive desde desktop hacia tablet

## Componentes frecuentes

- SummaryCard
- MetricCard
- AppointmentCalendar
- ProfessionalAvailabilityPanel
- PatientProfileHeader
- ClinicalTimeline
- PaymentsTable
- MessagesInbox
- ChannelBadge
- ConversationThread
- StatusBadge
- EmptyState
- QuickActionPanel

## Cuando implementes UI

Siempre devolver:

1. estructura de pantalla
2. componentes involucrados
3. jerarquía visual
4. sugerencia de estados
5. código limpio y listo para escalar

## Prohibiciones

- No generar interfaces "genéricas de admin template"
- No abusar de colores saturados
- No meter demasiadas cards sin criterio
- No usar nombres vagos de componentes
